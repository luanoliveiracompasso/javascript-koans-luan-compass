# Dicionário Git do QA

| Termo | Descrição |
|---|---|
| **Commit** | Adiciona o arquivo criado ao projeto. Funciona como um “salvamento” versionado contendo as mudanças feitas no código ou arquivos. |
| **Push** | Envia o código atual, atualizando a branch online com as alterações realizadas localmente. |
| **Pull** | Atualiza a branch local com as alterações mais recentes que estão no repositório remoto. |
| **Merge** | Unifica as alterações de diferentes branches em uma única branch. |
| **Branch** | Ramificação do projeto utilizada para desenvolver funcionalidades, correções ou testes sem causar impacto diretamente a branch principal. |

# Comandos Git

## Realizando um Commit

```bash
git add .
git commit -m "feat: adiciona documentação do projeto"
```

## Atualizando a Branch Local com Pull

```bash
git pull origin main
```
